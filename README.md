# Lost-and-Found-System
*Campus Lost & Found* is a lightweight, frontend-only web application designed to help students report and recover lost items on campus, replacing cluttered WhatsApp groups with a clean, searchable platform. Built using HTML, CSS, and vanilla JavaScript, the entire app runs in a single file with no dependencies or build steps required. Users can submit lost or found items with details like title, category, location, date, description, and an image, all of which are stored locally in the browser using localStorage so the app works completely offline. Key features include live filters for type, category, and location, instant text search, and smart match detection that alerts users when lost and found items share the same category and location. To encourage returns, the app includes a Campus Coins gamification system and one-click contact options via email or WhatsApp directly from each item card. This version was intentionally built as a frontend MVP to validate the idea quickly with zero hosting cost and instant deployment, while the codebase is structured to easily integrate a Firebase backend for shared, campus-wide data in v2.
